# 极端低成本的静态PWA视频站

现在基本前端UI页也有了，数据json基本在 \_by_id 和 \_by_tags 目录下，我们要做个支持者70k+的视频pwa网站，希望部署到 cloudflare
用无服务器架构，m3u8保存到r2, 用户登录数据存客户端本地，评论数据做静态的，不允许用户评论先，用户数据完全不上传，比如喜欢，播放记
录等，默认直接注册成本分配一个用户名和UID，还有个默认avatar,
只有最后开通会员的时候，才收集他的email和开通会员时间，时长，付费金额。 帮我完善下架构，实现github actions 自动部署到 cloudflare
，本地可以测试前后端，接口测试支持TDD，推荐算法就用最简单的 最新是最新，推荐是热门，视频详情页的相关推荐就固定4个
分配机会需要尽量全覆盖到所有ID都有机会在别的视频详情页下出现。 尽量用静态 json存r2来解决减少服务器运算压力，首页的分类标签下的分
页加载也是固定的顺序，我们不考虑更新，也没有搜索功能，抖音刷的信息流也是固定的分页加载，只要能支持用户一直加载更多页看下去即可。
